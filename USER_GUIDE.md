## 1. Обзор системы

**Deploy Agent** - это автоматизированная система деплоя git-репозиториев на различные платформы (Vercel, Netlify, Cloudflare Pages и др.) с полной поддержкой конфигурации.

### Ключевые особенности:
- 🚀 **Один клик деплой** из GitHub, GitLab или Bitbucket
- 🎯 **Поддержка множества платформ**: Vercel, Netlify, Cloudflare Pages, Railway
- 🔐 **Шифрование данных** (AES-256-GCM для переменных окружения)
- ⚡ **Retry с экспоненциальной задержкой** для всех внешних API
- 📊 **Система анализа репозиториев** и рекомендации хостинга
- 🎪 **Маркетплейс ревью** - тестирование проектов за баллы
- 📈 **Аналитика и дашборды** с экспортом данных

### Технологический стек:
- **Frontend**: Next.js 16 (App Router) + TailwindCSS + TypeScript
- **Backend**: Next.js API Routes + Node.js сервисы
- **База данных**: PostgreSQL (через Drizzle ORM)
- **Очереди/Кэш**: Redis (Upstash совместимый)
- **Аутентификация**: NextAuth (JWT токены)

## 3. Быстрый старт

### 3.1 Регистрация и вход
1. Откройте http://localhost:3000/landing
2. Нажмите **"Get Started"** или **"Sign Up"**
3. Введите email, пароль, имя
4. Подтвердите email (если требуется)
5. Войдите в систему

### 3.2 Создание первого проекта
1. Перейдите на http://localhost:3000/dashboard
2. Нажмите **"New Project"**
3. Введите URL репозитория (например, `https://github.com/vercel/next.js`)
4. Выберите платформу деплоя (по умолчанию Vercel)
5. Настройте параметры (опционально)
6. Нажмите **"Create Project"**

### 3.3 Первый деплой
1. В списке проектов нажмите **"Deploy"** напротив проекта
2. Дождитесь завершения деплоя
3. Перейдите по сгенерированному URL для проверки

## 4. Модуль деплоя

### 4.1 Страница проектов (`/projects`)

**Функции:**
- Список всех проектов пользователя
- Кнопка "New Project" для создания
- Статус каждого проекта (Active, Building, Error)
- Быстрые действия: Deploy, Settings, Delete

**Параметры проекта:**
| Параметр | Описание | По умолчанию |
|----------|---------|-----------------|
| `repo_url` | URL git-репозитория | - |
| `platform` | Платформа (vercel, netlify, cloudflare-pages, railway) | vercel |
| `environment` | Среда (production, preview, development) | production |
| `branch` | Ветка для деплоя | main |
| `root_directory` | Корневая директория | / |
| `build_command` | Команда сборки | - |
| `output_directory` | Директория выхода | - |

### 4.2 Настройка переменных окружения (`/projects/[id]/env`)

**Возможности:**
- Добавление/редактирование переменных
- Шифрование секретных данных (AES-256-GCM)
- Массовый импорт из .env файла
- Удаление переменных

**API:**
```bash
# Получить переменные
GET /api/projects/[id]/env-vars

# Добавить переменную
POST /api/projects/[id]/env-vars
{
  "key": "API_KEY",
  "value": "secret123",
  "is_secret": true
}

# Удалить переменную
DELETE /api/projects/[id]/env-vars/[varId]
```

### 4.3 Деплой и логи (`/deploy`, `/deploy/[id]`)

**Процесс деплоя:**
1. Нажатие "Deploy" в дашборде
2. Отправка запроса на Vercel API
3. Мониторинг через WebSockets (логи в реальном времени)
4. Получение URL развернутого проекта

**Просмотр логов:**
- Real-time логи через WebSockets
- Статус деплоя (pending → building → ready/error)
- Время сборки и детали ошибок

**API:**
```bash
# Создать деплой
POST /api/deploy
{
  "repo_url": "https://github.com/user/repo",
  "project_name": "my-project"
}

# Получить статус деплоя
GET /api/deploy?deployment_id=xxx

# Список деплоев проекта
GET /api/deploy?project_id=123
```

## 6. Маркетплейс ревью (NEW!)

### 6.1 Обзор системы

**Маркетплейс ревью** - это платформа, где:
- 👤 **Авторы проектов** отправляют свои проекты на тестирование
- 🧪 **Тестировщики** проверяют проекты и пишут отзывы
- ⭐ **Авторы оценивают качество** отзывов (1-5 звезд)
- 💰 **Тестировщики получают баллы** за качественные ревью
- 🏆 **Рейтинг** определяет лучших тестировщиков

### 6.2 Отправка проекта на ревью (`/projects/submit-for-review`)

**Процесс:**
1. Перейдите на http://localhost:3000/projects/submit-for-review
2. Выберите проект из списка
3. Выберите категорию:
   - SaaS (SaaS приложение)
   - E-commerce (Интернет-магазин)
   - Blog (Блог)
   - Portfolio (Портфолио)
   - Web App (Веб-приложение)
   - Mobile App (Мобильное приложение)
   - API (API сервис)
   - Dashboard (Дашборд)
   - Landing Page (Лендинг)
   - Other (Другое)
4. Заполните название и описание
5. Укажите URL живого демо (опционально)
6. Добавьте данные для входа (для тестировщиков)
7. Напишите инструкции по тестированию
8. Установите награду за ревью (баллы, по умолчанию 100)
9. Укажите максимальное количество тестировщиков (по умолчанию 5)
10. Нажмите **"Submit for Review"**

**API:**
```bash
POST /api/projects/submit-for-review
{
  "projectId": 123,
  "category": "web-app",
  "title": "My Awesome App",
  "description": "A great web application",
  "liveUrl": "https://my-app.vercel.app",
  "demoCredentials": {"username": "test", "password": "test123"},
  "testingInstructions": "Test the login flow",
  "pointsReward": 100,
  "maxTesters": 5
}
```

### 6.3 Просмотр проектов для ревью (`/review`)

**Функции:**
- Список всех проектов, доступных для тестирования
- Фильтрация по категориям
- Поиск по названию
- Отображение награды за ревью
- Статус текущего количества тестировщиков

**Процесс тестирования:**
1. Найдите интересный проект
2. Нажмите **"Review"**
3. Перейдите на страницу `/review/[submissionId]`
4. Протестируйте проект
5. Заполните форму отзыва:
   - Рейтинг (1-5 звезд)
   - Заголовок отзыва
   - Подробный контент
   - Найденные баги (количество)
   - Скриншоты (загрузка)
   - Чеклист тестирования (функциональность, UI, производительность, безопасность, документация)
   - Видео URL (опционально)
   - Затраченное время
6. Отправьте отзыв

**API:**
```bash
# Список проектов для ревью
GET /api/projects/submit-for-review?category=web-app&status=approved

# Отправить ревью
POST /api/reviews
{
  "submissionId": 456,
  "rating": 5,
  "title": "Great app!",
  "content": "Thoroughly tested...",
  "bugsFound": 2,
  "screenshots": ["base64encoded..."],
  "testingChecklist": {
    "functionality": true,
    "ui": true,
    "performance": false,
    "security": true,
    "documentation": false
  }
}
```

### 6.4 Оценка качества ревью (`/api/reviews/rate`)

**Как это работает:**
1. Автор проекта получает уведомление о новом отзыве
2. Автор изучает отзыв
3. Оценивает качество (1-5 звезд):
   - ⭐⭐⭐⭐⭐⭐ (5/5) = 100 баллов базовых
   - ⭐⭐⭐⭐ (4/5) = 50 баллов
   - ⭐⭐⭐ (3/5) = 30 баллов
   - ⭐⭐ (2/5) = 10 баллов
   - ⭐ (1/5) = 5 баллов
4. Бонусы:
   - +20 баллов за скриншоты
   - +15 баллов за заполненный чеклист
   - +10 баллов, если рейтинг отзыва ≥ 4
5. Баллы начисляются тестировщику

**API:**
```bash
POST /api/reviews/rate
{
  "reviewId": 789,
  "qualityScore": 5,
  "feedback": "Excellent review with screenshots!"
}
```

### 6.5 Рейтинг и лидерборд (`/leaderboard`)

**Типы рейтинга:**
- **Общий** (все категории вместе)
- **По категориям** (отдельно для каждой категории)

**Отображение:**
- Топ-50 тестировщиков
- Ваше место в рейтинге (выделено)
- Баллы, уровень, количество ревью
- Средний рейтинг и качество отзывов

**Система уровней:**
- Level 1: 0-999 баллов
- Level 2: 1000-1999 баллов
- Level 3: 2000-2999 баллов
- И так далее...

### 6.6 Профиль пользователя (`/profile`)

**Информация:**
- **Баллы**: Всего, доступно, потрачено
- **Уровень**: Текущий уровень + прогресс-бар
- **Транзакции**: История начислений и трат
- **Рейтинг**: Место в общем и по категориям

**Вкладки:**
- **Overview**: Быстрая статистика
- **Transactions**: История транзакций
- **Rankings**: Ваши места в рейтингах

## 10. Настройка и конфигурация

### 10.1 Переменные окружения (.env)

**Обязательные:**
```env
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/deploy_agent
VERCEL_TOKEN=vercel_xxxxxxxx
GITHUB_TOKEN=ghp_xxxxxxxx
ENCRYPTION_KEY=my-32-character-encryption-key!!
```

**Опциональные:**
```env
REDIS_URL=redis://localhost:6379  # Или Upstash URL
JWT_SECRET=same-as-encryption-key
PORT=3000
NODE_ENV=development
```

### 10.2 Настройка платформ

**Vercel (по умолчанию):**
- Требуется `VERCEL_TOKEN`
- Получите на https://vercel.com/account/tokens

**Netlify:**
- Добавьте `NETLIFY_TOKEN` в .env
- Получите на https://app.netlify.com/user/applications

**Cloudflare Pages:**
- Использует `VERCEL_TOKEN` (совместимый API)
- Настройте в параметрах проекта

**Railway:**
- Добавьте `RAILWAY_TOKEN`
- Получите в настройках аккаунта Railway

### 10.3 Лимиты и ограничения

**Rate Limiting (защита от спама):**
```env
RATE_LIMIT_WINDOW_MS=60000  # Окно (1 минута)
RATE_LIMIT_MAX_REQUESTS=10   # Максимум запросов
```

**Лимиты биллинга (зависят от плана):**
- Free: 3 проекта, 10 деплоев/день
- Pro: 20 проектов, 100 деплоев/день
- Team: ∞ проектов, 1000 деплоев/день

## 11. Частые вопросы

### 11.1 Общие вопросы

**В: Как сбросить пароль?**
О: В настоящее время сброс пароля недоступен. Обратитесь к администратору.

**В: Можно ли удалить проект?**
О: Да, в настройках проекта (`/projects/[id]`). Это удалит проект и все деплои.

**В: Как изменить платформу деплоя?**
О: Настройки проекта → Platform → Выберите новую платформу → Save.

### 11.2 Маркетплейс ревью

**В: Как получить баллы?**
О: Отправляйте качественные ревью проектов. Авторы проектов оценивают качество (1-5 звезд), и вы получаете баллы.

**В: Сколько баллов за ревью?**
О: Зависит от оценки качества:
- 5/5 = 100 баллов (базовых) + бонусы
- 4/5 = 50 баллов + бонусы
- 3/5 = 30 баллов
- 2/5 = 10 баллов
- 1/5 = 5 баллов

**В: Бонусы за что?**
О: +20 за скриншоты, +15 за заполненный чеклист, +10 если рейтинг ≥ 4.

**В: Как попасть в лидерборд?**
О: Получите хотя бы 1 балл за ревью, и вы появитесь в рейтинге.

**В: Можно ли тратить баллы?**
О: Да! В будущих обновлениях баллы можно будет тратить на премиум-функции (приоритетный деплой, расширенная аналитика и др.).

### 11.3 Технические вопросы

**В: Ошибка "DATABASE_URL is invalid"**
О: Проверьте, что PostgreSQL запущен и URL корректный:
```bash
psql "postgresql://postgres:postgres@localhost:5432/deploy_agent" -c "SELECT 1;"
```

**В: Ошибка "VERCEL_TOKEN is invalid"**
О: Убедитесь, что токен:
- Не истек
- Имеет правильный формат (`vercel_xxx`)
- Имеет необходимые права (Full Account Access)

**В: `npm run db:push` зависает**
О: Выполните SQL вручную:
```bash
cat drizzle/0002_review_marketplace.sql | psql -U postgres -d deploy_agent
```

**В: Билд падает с ошибкой TypeScript**
О: Убедитесь, что используете Node.js >= 18:
```bash
node --version  # должно быть >= 18.0.0
```

### 11.4 Администрирование

**В: Как сделать пользователя админом?**
О: Выполните SQL:
```sql
UPDATE users SET is_admin = true WHERE email = 'user@example.com';
```

**В: Как посмотреть логи?**
О: Логи выводятся в консоль при `npm run dev`.
Для продакшена настройте логирование в файл.

**В: Как очистить базу данных?**
О:
```bash
dropdb deploy_agent
createdb deploy_agent
npm run db:push
```
