## Введение

**Deploy Agent** — это платформа для автоматического деплоя приложений из Git-репозиториев на различные хостинг-платформы (Vercel, Netlify, Railway и др.) с поддержкой полной настройки параметров.

### Основные возможности
- Деплой из GitHub, GitLab, Bitbucket одним кликом
- Поддержка множества платформ: Vercel, Netlify, Cloudflare Pages, Railway
- Полная настройка через API или интерфейс (100+ параметров)
- Шифрование переменных окружения (AES-256-GCM)
- Ограничение частоты запросов (Rate Limiting)
- Автоматический повтор при сбоях (Retry с экспоненциальной задержкой)
- Интеграция с GitHub Actions для CI/CD
- Логирование деплоев в реальном времени
- Автоматизация Supabase

### Новые возможности (май 2026)
- Генератор лендингов на базе ИИ (OpenAI)
- Реферальная система и партнерская программа
- Биллинговая система на базе Stripe
- Система анализа репозиториев
- Маркетинговые инструменты и аналитика
- Маркетплейс код-ревью и тестирования

## Для пользователей

### Регистрация и вход

1. Перейдите на главную страницу `/`
2. Нажмите "Sign Up" для регистрации или "Login" для входа
3. После входа вы попадете в личный кабинет `/dashboard`

### Личный кабинет (Dashboard)

**URL:** `/dashboard`

В личном кабинете доступны:
- Список ваших проектов
- Статистика деплоев
- Быстрые действия (создать проект, просмотреть аналитику)
- Ссылка на реферальную программу

### Создание нового проекта

1. Перейдите на `/projects/new`
2. Введите URL репозитория (например: `https://github.com/user/repo`)
3. Выберите платформу для деплоя:
   - **Vercel** — рекомендуется для Next.js проектов
   - **Netlify** — для статических сайтов и JAMstack
   - **Railway** — для полноценных приложений с БД
   - **Cloudflare Pages** — для статического хостинга
4. Настройте параметры (опционально):
   - Ветка (по умолчанию: `main`)
   - Корневая директория (по умолчанию: `/`)
   - Команда сборки
   - Переменные окружения
5. Нажмите "Deploy" — деплой начнется автоматически

### Просмотр хостинг-предложений (партнерская программа)

**URL:** `/partners`

На этой странице представлены предложения от партнеров:
- Vercel — бесплатный хостинг для фронтенда
- Netlify — JAMstack платформа
- Railway — хостинг с базой данных

Каждое предложение содержит:
- Описание платформы
- Реферальную ссылку (вы получаете комиссию при регистрации по ней)
- Кнопку "View Offer" для перехода

### Управление проектами

**URL:** `/projects`

В списке проектов доступны действия:
- Просмотр статуса деплоя
- Просмотр логов (`/deploy/[id]`)
- Управление переменными окружения (`/projects/[projectId]/env`)
- Редактирование настроек проекта
- Удаление проекта

### Настройки профиля

**URL:** `/settings`

Доступные разделы:
- **Profile** — редактирование имени, email
- **Tokens** — управление API-токенами (GitHub, Vercel, Netlify)
- **Supabase** — настройка интеграции с Supabase
- **Subscription** — управление тарифным планом (см. раздел "Биллинг")

### Биллинг и подписка

**URL:** `/billing`

#### Тарифные планы

| План | Цена | Возможности |
|------|-------|-------------|
| **Free** | $0 | 3 проекта, 10 деплоев/день, 1 пользователь |
| **Pro** | $29/мес | 20 проектов, 100 деплоев/день, 5 участников команды |
| **Team** | $99/мес | Безлимит проекты, 1000 деплоев/день, 50 участников, SSO |
| **Enterprise** | $499+/мес | Все возможности, выделенный инстанс, кастомный SLA |

#### Управление подпиской
1. Выберите план на странице `/billing`
2. Нажмите "Upgrade" — вы будете перенаправлены на Stripe Checkout
3. Оплатите подписку
4. После оплаты лимиты обновятся автоматически

#### Customer Portal
Для управления подпиской (отмена, смена карты): нажмите "Manage Subscription" на странице биллинга.

### Реферальная программа

**URL:** `/referrals`

#### Как это работает
1. У каждого пользователя есть уникальный реферальный код
2. Поделитесь ссылкой с друзьями: `https://yourapp.com/?ref=ВАШ_КОД`
3. Когда кто-то регистрируется по вашей ссылке — вы получаете комиссию
4. Статистика отображается в личном кабинете

#### Отслеживание
- **Клики** — сколько раз перешли по вашей ссылке
- **Конверсии** — сколько человек зарегистрировались
- **Комиссия** — заработанные средства

### Маркетплейс код-ревью

#### Отправка проекта на ревью

**URL:** `/projects/submit-for-review`

1. Выберите проект из списка
2. Выберите категорию (SaaS, E-commerce, Blog, Portfolio и др.)
3. Опишите проект
4. Отправьте на ревью

#### Прохождение ревью (как тестировщик)

**URL:** `/review`

1. Просмотрите список проектов, ожидающих ревью
2. Выберите проект
3. Протестируйте его
4. Напишите отзыв с оценкой (1-5 звезд)
5. Прикрепите скриншоты, опишите баги
6. Получите баллы за качественный ревью

#### Система баллов
- Ревью с оценкой 5/5 → 100 баллов
- Ревью с оценкой 4/5 → 50 баллов
- Ревью с оценкой 3/5 → 30 баллов
- Бонусы: +20 за скриншоты, +15 за чеклист тестирования

#### Лидерборд

**URL:** `/leaderboard`

Просмотр рейтинга участников:
- Общий рейтинг
- По категориям
- Ваша позиция

### Стартап-инструменты

#### Шаблоны проектов

Готовые конфигурации для:

### Аналитика и экспорт

#### Просмотр аналитики
- Количество деплоев

### Публичные лендинги (AI-генерация)

## Для администраторов

### Вход в админ-панель

**URL:** `/admin`

Доступ только для пользователей с флагом `isAdmin=true`.

### Главная страница админ-панели

**URL:** `/admin`

Включает:
- Общую статистику (пользователи, проекты, деплои)
- Финансовую аналитику (MRR, конверсия, планы)
- Мониторинг здоровья системы
- Последние аудит-логи
- Уведомления

### Генератор лендингов на базе ИИ

**URL:** `/admin/marketing/generate-landing`

#### Генерация нового лендинга
1. Нажмите "+ Generate New"
2. Заполните форму:
   - **Topic** (тема) — например: "Launch your AI startup in 30 minutes"
   - **Target Audience** (целевая аудитория) — "AI enthusiasts who built a prototype"
   - **Tone** (тон) — Inspiring, Professional, Casual, Urgent
   - **Sections** (секции) — hero, features, cta, testimonials, pricing
3. Нажмите "Generate with AI"
4. ИИ (OpenAI gpt-4o-mini) сгенерирует JSON-конфигурацию
5. Предпросмотр лендинга появится сразу под формой

#### Публикация лендинга
1. Введите slug (например: `ai-startup-launch`)
2. Нажмите "Publish Page"
3. Лендинг станет доступен по адресу `/landing/[slug]`
4. В базе данных установится флаг `isPublished=true`

#### Управление существующими лендингами
- **View** — просмотр опубликованного лендинга
- **Publish** — публикация черновика
- **Delete** — удаление лендинга

#### API эндпоинты
| Method | Endpoint | Описание |
|--------|----------|-----------|
| POST | `/api/admin/generate-landing` | Генерация конфигурации через OpenAI |
| POST | `/api/admin/generate-landing/publish` | Публикация лендинга |
| POST | `/api/admin/generate-landing/delete?id=ID` | Удаление лендинга |

#### Обязательные переменные (.env)

```bash

# Vercel

# GitHub

# Шифрование
ENCRYPTION_KEY=change-this-32-character-encryption-key

# OpenAI (для генерации лендингов)
OPENAI_API_KEY=sk-your_openai_api_key_here

# Stripe (для биллинга)
STRIPE_SECRET_KEY=sk_test_your_stripe_secret_key_here
STRIPE_WEBHOOK_SECRET=whsec_your_webhook_secret_here
STRIPE_PRICE_PRO_ID=price_your_pro_plan_price_id
STRIPE_PRICE_TEAM_ID=price_your_team_plan_price_id
STRIPE_PRICE_ENTERPRISE_ID=price_your_enterprise_plan_price_id
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_test_your_publishable_key_here

# URL приложения
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

### Безопасность

#### Меры безопасности
- Все чувствительные данные (API ключи, пароли) шифруются алгоритмом AES-256-GCM
- Ограничение частоты запросов (10 запросов/минуту по умолчанию)
- Переменные окружения никогда не логируются
- Проверка подписи вебхуков (Stripe)
- JWT аутентификация с HTTP-only cookies

#### Stripe Webhooks
Для корректной работы биллинга настройте вебхук в Stripe Dashboard:
- URL: `https://yourdomain.com/api/webhooks/stripe`
- События: `checkout.session.completed`, `invoice.payment_succeeded`, `customer.subscription.deleted`

## Часто задаваемые вопросы (FAQ)

### Как получить API токены?

**Vercel Token:**
1. Перейдите на https://vercel.com/account/tokens
2. Создайте новый токен
3. Скопируйте и вставьте в `.env` файл

**GitHub Token:**
1. Перейдите в настройки GitHub → Personal Access Tokens
2. Создайте токен с правами `repo`
3. Скопируйте и вставьте в `.env`

**OpenAI API Key:**
1. Зарегистрируйтесь на https://platform.openai.com
2. Создайте API ключ в разделе API Keys
3. Скопируйте и вставьте в `.env`

### Почему деплой не работает?

Проверьте:
1. Правильность API токенов в `.env`
2. Доступность репозитория (публичный или токен имеет доступ)
3. Лимиты вашего тарифного плана (количество деплоев/день)
4. Логи деплоя на странице `/deploy/[id]`

### Как стать администратором?

Установите флаг `isAdmin=true` в базе данных:
```sql
UPDATE users SET is_admin = true WHERE email = 'your_email@example.com';
```

### Как настроить Stripe для продакшна?

1. Создайте аккаунт на https://stripe.com
2. Настройте продукты и цены (Price IDs)
3. Настройте вебхук в Stripe Dashboard
4. Добавьте все ключи в `.env`
5. Протестируйте на тестовых ключах перед запуском
